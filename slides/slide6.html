Operating Systems Security - Core Concepts

Role of Operating Systems in Security

  • Controls interactions between users, applications, and hardware.
  • Provides user authentication, access control, process isolation, and resource protection.

Threats to Operating Systems

  • Malicious code (viruses, worms).
  • Unauthorized access and privilege escalation.
  • Buffer overflow, denial of service (DoS) attacks.
  • Insider threats and system misconfiguration.

Process Organization

  • Processes run in isolated memory spaces.
  • User mode vs Kernel mode: Kernel mode has full system access.

Technical Attack Types

Buffer Overflow Attack

  • Occurs when more data is written to a buffer than it can hold.
  • Can overwrite adjacent memory, allowing attackers to control execution flow.

String Formatting Attack

  • Occurs when user input is directly passed to formatted output functions like printf().
  • Can leak memory addresses or crash programs.

Integer Overflow Attack

  • Occurs when arithmetic operations exceed the maximum value of an integer.
  • Can bypass security checks, leading to vulnerabilities.

Code Injection Attack

  • Attacker injects and executes malicious code inside vulnerable applications.
  • Examples: SQL Injection, Cross-Site Scripting (XSS).

Protection Techniques

  • Address Space Layout Randomization (ASLR): Randomizes memory locations.
  • Stack Canaries: Detects buffer overflows before overwriting return addresses.
  • Non-Executable Memory (NX Bit): Marks memory regions as non-executable.
  • Code Reviews: Find vulnerabilities like overflows early.
