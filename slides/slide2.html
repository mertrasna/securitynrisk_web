Slide 2: Cryptography I - Core Ideas

What is Cryptography?

  • It's the study of mathematical techniques to protect information
  • It provides important security services like:
    • Confidentiality (hide the information)
    • Integrity (ensure no one changed it)
    • Authentication (verify who created/sent the info)
    • Authorization (allow access based on identity)
    • Non-repudation (prove who sent it - can‚Äôt deny it later)

What is a Cryptosystem?

  • It's basically a formal way to define how encryption works, using 5 components:
    • P - Set of plain messages (normal text)
    • C - Set of cipher messages (encrypted text)
    • K - Set of keys (secret values)
    • E - Encryption function (turns P into C)
    • D - Decryption function (turns C back into P)
  • There's an important property:
    • Dk(Ek(m)) = m
    • It means: if you encrypt a message with key "k" and then decrypt it with the same key, you get back the original message!

Kerckhoff's Principle

  • The system's security should depend only on the secrecy of the key, not on the secrecy of the encryption algorithm itself.
  • Example: Even if an attacker knows you're using AES encryption, they still can't decrypt your message without the key.

Visual example to remember: Think of Cryptography like sending a locked treasure chest

  • Message (P) = treasure inside
  • Lock (E) = encrypts the treasure
  • Key (K) = key to unlock it
  • Chest when locked (C) = cipher text
  • Unlocking (D) = decryption to get the treasure back!
