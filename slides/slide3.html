Security and Risk

RSA (Rivest-Shamir-Adleman Algorithm)

  • A type of asymmetric encryption (uses two different keys).
  • Public key (PU) encrypts, Private key (PR) decrypts.
  • Based on very hard math problem: factoring large primes
  • Steps:
    • 1. Choose two large prime numbers "p" and "q"
    • 2. Compute "n = p * q"
    • 3. Choose e (public exponent, often 65537)
    • 4. Find d (private exponent) using special math
  • Encryption: C = M^e mod n
  • Decryption: M = C^d mod n
  • Public key (e,n) -> for encrypting
  • Private key (d,n) -> for decrypting

Diffie-Hellman Key Exchange

  • Securely exchange a secret key over insecure channels
  • Based on difficulty of discrete logarithm problem
  • Steps:
    • 1. Agree on public values: g (generator) and p (large prime)
    • 2. Alice chooses secret "a", Bob chooses secret "b"
    • 3. Alice sends "g^a mod p" to Bob, Bob sends "g^b mod p" to Alice
    • 4. Both compute the same shared key:
      • Alice computes "(g^b mod p)^a mod p"
      • Bob computes "(g^a mod p)^b mod p"
  • They both get the same result without sharing their secrets!

Digital Signatures

  • Used for:
    • Authenticating sender
    • Proving message integrity
    • Preventing sender from denying ("Non-repudiation")
  • How it works:
    • 1. Sender signs the message using private key
    • 2. Receiver verifies the signature using public key
  • Used in secure emails, contracts, financial transactions, etc.

Fast Summary

  • RSA -> Public key encryption, private key decryption
  • Diffie-Hellman -> Create shared key without exposing secrets
  • Digital Signatures -> Prove authenticity + integrity with private key signature
