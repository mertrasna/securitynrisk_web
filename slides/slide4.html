Access Control - Core Concepts

What is Access Control?

  • Access Control = Who can access which resource under which conditions.
  • It's way to protect systems and information from unauthorized access.

Main Steps in Access Control:

  • Identification - Who are you? (username, ID)
  • Authentication - Can you prove it? (password, biometric, token)
  • Authorization - What are you allowed to do? (permissions, access level)
  • Accountability - Can we track what you did? (logs, audit trails)

Identity Management (IdM)

  • Managing user identities automatically:
    • Create, update, and delete accounts
    • Password resets
    • Single Sign-On (SSO)

Password Attacks:

  • Brute Force: Try all possible passwords.
  • Dictionary Attacks: Try common words first.
  • Rainbow tables: Precomputed hashes of passwords.
  • Social engineering: Trick people into giving their passwords.

Password Protection Techniques

  • Hashing passwords instead of storing them directly.
  • Salting: Add random data to passwords before hashing.
  • Shadow files: Store hashed passwords in protected files.

Access Control Models:

  • Mandatory Access Control (MAC) - System enforces strict rules
  • Discretionary Access Control (DAC) - Owners decide who can access their resources
  • Role-Based Access Control (RBAC) - Access based on user roles (admin, manager, etc.)
  • Attribute-Based Access Control (ABAC) - Access based on attributes (age, department, etc.)

Famous Security Models:

  • Bell-LaPadula Model (confidentiality focused):
    • No read up (simple security property)
    • No write down (*-property)
  • Biba Model (integrity focused):
    • No write up
    • No read down
