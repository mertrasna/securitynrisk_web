Bad Networks - Core Concepts

What are "Bad Networks"?

  • Communication infrastructures used by malicious actors (e.g., botnets).
  • Networks built to perform attacks, spread malware, or hide attacker operations.

Botnets

  • Network of compromised computers (bots) controlled by an attacker (botmaster).
  • Used for DDoS attacks, spam campaigns, data theft, and mining cryptocurrency.
  • Control often done through Command and Control (C2) servers.

Communication Methods in Bad Networks

  • Centralized Control: Single C2 server commands all bots (easy to take down).
  • Peer-to-Peer (P2P): Bots communicate directly (harder to disrupt).
  • Decentralized Control: Distributed systems with multiple communication nodes.

Evasion Techniques

  • Encryption: Hides communications from detection tools.
  • Fast-Flux DNS: Rapidly changing IP addresses to avoid blocking.
  • Domain Generation Algorithms (DGAs): Generate random domain names for bot communication.
  • Use of legitimate services (like Dropbox, Twitter) to hide malicious traffic.
